- **랩어라운드(wrap-around)**
    - 360도에서 0으로 각도가 변경되는 구조

- 프로래머스 Lv2 PCCP 기출문제 3번-아날로그 시계 문제
    - 해당 문제의 난이도 상승 원인
        - 초당 시침의 이동 각도가 약 0.0083이라서 처음 작동하던 방식이던  가중치 더하기 방식이 제대로된 값이 아니었음
    - 가중치 더하기 방식에서 변경한 방식
        - 초당 가중치 더함에서 초당 새 계산 값을 사용
        - 기존에는 double 자료형에서 바로 덧셈하였지만 변경 후, BigDecimal 자료형을 사용해서 정확성을 높임
        - 또한 시, 분, 초로 분리하여 계산하던 방법에서 그냥 전부 초로 변환한 후 순환 인덱스를 더하여 초 기준으로 계산해서 값을 산출
        - 그리고 360도는 0도로 변경되기 때문에 값 비교에서 오류가 발생하여 만약 이전 값보다 이후 값이 더 작다면 360을 더해서 보정값을 생성해 그것으로 비교
            - 이전 값이 이후 값보다 크다는 의미는 예를 들어 이전에는 355도 였는데 이후에는 10도가 되었다는 의미로 실제로는 10도에 해당하는 침이 더 큰 값을 가지지만 숫자로 보면 355도에 있었던 때가 더 크다고 판단되므로 360도를 더해서 보정값을 만듦
    - 문제의 핵심
        - 초침이 시/분침과 일치한다고 문제에선 나왔지만 실제론 이것들을 추월했을 때의 의미를 가짐
        - 문제에 나와 있지 않은 시/분/초침의 초당 이동 각도를 알아야함 이 정보는 내 생각엔 문제에서 고지를해야하지만 이를 설명하지 않아 문제를 푸는 사람이 직접 검색을 통해 알아야했음
    - 이 문제의 목적
        - 랩어라운드 구조를 이해해야 하며 부동소수점 게산은 항상 정확하지 않다는 것을 이해해야한다.
