- 프로래머스 Lv3 브라이언의 고민 문제
    - 해당 문제의 난이도 상승 원인
        - 너무나 많은 반례들이 있고, 그 모든 반례들을 전부 만족하기가 힘들었다.
    - 해당 문제를 해결한 방법
        - 해당 문제는 결국 해결하지 못하였슴
    -  처음 방식에서 변경한 방식
        - 처음
            - 들어온 문자열을 순환하며 규칙1, 규칙2의 처리법을 따르게한 후에 인덱스 이동
        - 변경
            - 처음 문자열에서 소문자의 위치를 파악 후, 기존의 방식과는 비슷하지만 오류 검사 부분을 추가
    - 문제의 핵심
        - 규칙1(단어 사이사이에 소문자 존재), 규칙2(단어 양끝에 소문자 존재)인 경우는 기본이요 중첩된 상태(규칙2가 적용된 단어에 규칙1까지 적용)까지 모두 고려하여 검사를 해야한다
        - 이미 사용한 소문자 검사를 필요
        - 특이 케이스(규칙1과 규칙2의 조건을 모두 충족한 경우)의 처리법(뒤에 상태를 검사하여 앞의 특이 케이스를 규칙1로 할 것인지 규칙 2로 할 것인지)
    - 이 문제의 목적
        - 보기에서 주어진 규칙1, 규칙2인 경우만이 아닌, 이 둘이 합쳐진 케이스를 처리하는 방법을 어떻게 만들 것인지에 대해 학습
