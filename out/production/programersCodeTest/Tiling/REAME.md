- 프로래머스 Lv2  3 * N 타일링문제
    - 해당 문제의 난이도 상승 원인
        - size가 커질 시, 순환문 사용으로 인한 과부하 처리
        - 배치 방법의 개수값이 기하급수 적으로 증가 시의 처리
        - 점화식을 통해 처리를 해야하는데 정작 내가 점화식을 모름
    - 해당 문제를 해결한 방법
        - 메모이제이션(Memoization) 방법을 통해 중복되는 순환 계산을 줄여 효율성 증가
    -  처음 방식에서 변경한 방식
        - 처음
            - 메모이제이션(Memoization)을 사용하지 않은 재귀함수를 통한 단순 순환 계산을 사용해 처리하여 하였으나 시간초과로 인해 변경이 필요함을 느낌
        - 변경
            - 메모이제이션(Memoization) 라는 것과 점화식을 사용해 중복되는 계산을 줄여 효율성 증가
    - 문제의 핵심
        - 블록배치 개수가 증가함에 따라 경우의 수가 일정하다는 것을 파악하는 것
    - 이 문제의 목적
        - 자료형 변환을 통해 리스되는 값 없이 처리하는 것
        - 재귀함수 사용 시, 메모이제이션(Memoization) 방법을 사용해 중복되는 계산을 줄이는 것
        - 점화식을 알고 있다면 사용해야 하며 점화식을 모른다면 학습 후 사용할 수 있도록 도움
        - 이전의 값을 사용할 필요가 없다면 **백트래킹/DFS**을, 중복을 허용한다면 **점화식/DP**
